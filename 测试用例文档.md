# LinkX 测试用例文档

## 一、测试用例清单

### 1. 认证模块测试用例

| 用例编号 | 用例名称 | 前置条件 | 测试步骤 | 预期结果 |
|---------|---------|---------|---------|---------|
| AUTH-001 | 正常登录 | 用户已注册 | 输入正确用户名密码，点击登录 | 登录成功，跳转主页，显示 Token |
| AUTH-002 | 错误密码登录 | 用户已注册 | 输入正确用户名+错误密码 | 提示"用户名或密码错误"，不跳转 |
| AUTH-003 | 未注册用户登录 | 用户未注册 | 输入未注册的账号密码 | 提示"用户名或密码错误" |
| AUTH-004 | 连续登录失败 | 无 | 连续 5 次输入错误密码 | 账号被锁定 15 分钟 |
| AUTH-005 | Token 刷新 | 持有有效 Refresh Token | Access Token 过期后访问 API | 自动刷新，返回新 Token |
| AUTH-006 | Refresh Token 失效 | Refresh Token 被撤销 | 使用失效的 Refresh Token | 返回 401，跳转登录页 |
| AUTH-007 | 注销登录 | 已登录 | 点击注销按钮 | 清除 Token，返回登录页 |
| AUTH-008 | 注册新用户 | 无 | 输入用户名、密码、昵称完成注册 | 注册成功，可登录 |

## 二、安全测试用例

| 用例编号 | 测试项 | 测试步骤 | 预期结果 |
|---------|-------|---------|---------|
| SEC-001 | SQL 注入 | 在搜索框输入 `' OR 1=1 --` | 不执行 SQL，返回空结果 |
| SEC-002 | XSS 攻击 | 在消息内容输入 `<script>alert(1)</script>` | 转义处理，不执行脚本 |
| SEC-003 | Token 伪造 | 使用假 Token 访问 API | 返回 401 未授权 |
| SEC-004 | 越权访问 | A 用户尝试操作 B 用户数据 | 返回 403 无权访问 |
| SEC-005 | 密码强度 | 注册时输入弱密码 | 提示密码强度不足 |
| SEC-006 | 文件上传 | 上传可执行文件 | 验证文件类型，拒绝上传 |

## 三、性能测试用例
