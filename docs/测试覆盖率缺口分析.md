# LinkX 测试覆盖率缺口分析

## 一、当前已有测试体系

### 1. 前端测试
前端已经配置：
- Vitest 单元测试
- 覆盖率统计
- Playwright E2E

可从以下脚本看出：
- test
- test:coverage
- test:e2e

### 2. 前端覆盖率配置
前端已设置覆盖率门槛：
- lines: 15%
- functions: 10%
- branches: 20%
- statements: 15%

说明前端不是空白状态，而是已经建立最低测试门禁。

### 3. 后端测试
后端已配置：
- spring-boot-starter-test
- H2 内存数据库
- embedded redis
- JaCoCo
- surefire
- 覆盖率门禁

后端覆盖率要求大约为：
- 行覆盖率 45%
- 指令覆盖率 40%

## 二、当前覆盖得较好的区域

### 1. 前端 API 层
已有大量 `api/*.test.ts`，说明以下内容覆盖较多：
- 请求参数构造
- 接口返回处理
- 错误处理
- 基础请求封装

### 2. 工具函数层
例如：
- fileConvert
- lockPin
- mediaUrl
- tokenStorage
- validation
- chatTime
- parseJson
- downloadFile

这类函数通常容易做高质量单测。

### 3. 核心 UI 组件
例如：
- LoginView
- ChatPanel
- allComponents.mount

说明关键 UI 不是完全没测。

### 4. 后端基础业务链路
后端已有一些偏流程型测试，例如：
- CallControllerTest
- CloudDriveControllerTest
- GroupInvitationControllerTest
- FriendChatFlowTest
- CallAndRedPacketFlowTest

说明后端测试已经开始从单点走向业务流程。

## 三、覆盖缺口最大的地方

### 1. IM 主链路端到端测试不足
这是最大缺口之一。

建议必须覆盖：
- 登录
- 拉取会话列表
- 打开会话
- 发送消息
- WebSocket 推送
- 消息落库
- 刷新后仍可恢复
- 重连后消息补偿

### 2. WebSocket 实时链路测试不足
IM 的核心不只是 REST，更是实时通信。

建议补：
- WebSocket 建连
- 心跳
- 重连
- 消息乱序
- 重复消息
- 离线补偿
- 服务端断连恢复

### 3. 异常路径覆盖不足
最容易漏，但最容易出线上问题：
- token 过期
- refresh token 失效
- 文件上传失败
- 断网
- Redis 异常
- 数据库超时
- MinIO 不可用
- 邮件服务异常

### 4. UI 复杂交互覆盖不足
IM 前端交互复杂，容易漏测：
- 表情面板
- 输入框快捷键
- 右键菜单
- 历史消息滚动加载
- 会话切换
- 未读角标
- 锁屏 / 解锁
- 主题切换
- 多窗口场景

### 5. 群聊与好友边界条件不足
边界场景特别重要：
- 拉黑后是否还能发消息
- 被踢出群后状态如何变化
- 群解散后会话如何处理
- 删除好友后历史消息保留规则
- 群公告修改后的通知范围
- 重复邀请 / 过期邀请 / 失效邀请

### 6. 文件与媒体测试不够细
企业级文件能力应补：
- 大文件上传
- 非法格式
- 过期链接
- 下载中断恢复
- 图片预览失败
- 语音播放失败
- 上传取消

### 7. 业务一致性测试不足
IM 项目常见问题是“单点成功，但状态不同步”：
- 会话未及时更新
- 未读数不一致
- 已读状态错乱
- 消息落库成功但通知失败
- 收藏 / 群文件 / 网盘引用数据不一致

## 四、覆盖率门禁现状解读

### 前端
当前门槛：
- 行覆盖率 15%
- 函数覆盖率 10%
- 分支覆盖率 20%
- 语句覆盖率 15%

这说明前端测试体系已建立，但整体覆盖水平还处在起步阶段。

### 后端
当前门槛：
- 行覆盖率 45%
- 指令覆盖率 40%

说明后端测试成熟度高于前端，但离高强度全面覆盖仍有空间。

## 五、优先补齐建议

### 第一优先级
1. WebSocket 实时链路测试
2. 登录 / 刷新 / 离线消息补偿测试
3. 发送消息全链路测试
4. 异常场景测试

### 第二优先级
5. 群聊管理测试
6. 文件上传 / 下载 / 预览测试
7. 通知 / 未读数 / 会话同步测试
8. 前端关键交互 E2E

### 第三优先级
9. 多端同步
10. 安全审计
11. 大群性能场景
12. 音视频通话稳定性测试

## 六、总结

### 企业 IM 标准
当前项目已经具备较完整的企业 IM 基础能力，但仍缺少：
- 消息可靠性闭环
- 已读 / 回执
- 撤回 / 编辑 / 转发
- 搜索
- 多端同步
- 更完整的安全审计
- 更细的群管理体系

### 测试覆盖
当前测试体系已经成型，但还不能称为全面覆盖，主要缺口在：
- IM 主链路端到端
- WebSocket 实时链路
- 异常路径
- 多端同步
- UI 复杂交互
- 业务一致性
