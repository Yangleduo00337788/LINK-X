# LinkX 整改优先级计划

## 一、目标

将 LinkX 从“具备 IM 基础能力”推进到“企业级可用、可验证、可持续迭代”的水平。

## 二、优先级划分

### P0：必须立即补齐
这些能力直接影响 IM 是否能稳定运行。

#### 功能
- 消息可靠性闭环
- 离线消息补偿
- 重连后消息同步
- 已读 / 未读 / 送达回执
- WebSocket 稳定链路
- 消息幂等与去重

#### 测试
- 登录 -> 发消息 -> 收消息 -> 落库 -> 刷新恢复
- WebSocket 连接 / 重连 / 心跳
- token 过期与刷新失败
- 异常断网恢复
- 消息重复与乱序

### P1：强烈建议补齐
这些能力会显著提升企业可用性。

#### 功能
- 消息撤回 / 编辑 / 转发
- 全文搜索
- 会话置顶 / 免打扰
- 群管理增强
- 文件大文件与断点续传
- 通知体系增强

#### 测试
- 撤回后状态一致性
- 搜索结果准确性
- 群成员权限边界
- 文件上传失败与恢复
- 消息通知与未读数同步

### P2：中长期增强
这些能力更偏成熟度和行业对标。

#### 功能
- 多端同步
- 在线设备管理
- 安全审计
- 敏感词 / 黑名单
- 音视频通话增强
- 超大群优化

#### 测试
- 多端登录与状态同步
- 踢下线场景
- 审计日志完整性
- 弱网下音视频稳定性
- 大群消息压力测试

## 三、推荐实施顺序

### 第 1 阶段
先补 P0：
- WebSocket 稳定性
- 消息可靠性
- 离线补偿
- 回执体系

### 第 2 阶段
再补 P1：
- 撤回 / 编辑 / 转发
- 搜索
- 群管理
- 文件体系
- 通知体系

### 第 3 阶段
最后补 P2：
- 多端同步
- 审计
- 合规
- 音视频增强
- 性能优化

## 四、验收标准建议

### 功能验收
- 消息链路完整闭环
- 群聊与单聊核心能力可用
- 文件 / 通知 / 搜索能力可验证

### 测试验收
- 核心链路有端到端测试
- 异常路径有覆盖
- 关键模块有覆盖率门禁
- E2E 冒烟可稳定运行

### 质量验收
- 核心业务改动不引发回归
- 消息状态一致性可保证
- 关键场景可重复验证

## 五、最终目标

将 LinkX 建成：
- 具备企业 IM 核心能力
- 有稳定的测试体系
- 具备持续迭代和回归保障
- 可作为内部协同平台长期演进的基础
