# LinkX 整改优先级计划（状态同步 2026-07-24）

## 一、目标

将 LinkX 建成：企业 IM 核心可用、可验证、可持续迭代的内部协同基础。

## 二、优先级与完成态

| 阶段 | 目标 | 状态 |
|------|------|------|
| P0 | 消息可靠性、WS、回执、主链路测试 | ✅ 完成 |
| P1 | 撤回/搜索/群/文件/通知与对应测试 | ✅ 完成 |
| P2 | 多端/审计/合规/音视频增强/大群优化 | ✅ 完成（含已知边界） |
| 质量门禁 | JaCoCo 关键模块 + CI 回归 + E2E | ✅ 完成 |

明细见 [`未完成清单.md`](./未完成清单.md)。剩余可选项见该文档「可选后续」。

## 三、推荐实施顺序（历史）

1. P0 → 2. P1 → 3. P2 —— **均已走完**。

新需求按「可选后续」或独立迭代排期，不再占用本计划 P0–P2 债务。

## 四、验收标准（当前口径）

### 功能
- 消息闭环、单聊/群聊核心、文件/通知/搜索可验证
- 重要会话高亮、搜索定位气泡可用

### 测试
- 核心链路 E2E / WS Live / 异常路径纳入 CI 回归子集
- 关键模块 JaCoCo：行 ≥45% / 指令 ≥40%
- Playwright：smoke + call-weaknet

### 质量
- 主路径改动有回归套件兜底
- 已知边界（mesh / FCM / 真压测）文档化，不混入「未完成」

## 五、最终目标（维持）

- 具备企业 IM 核心能力
- 有稳定测试与回归保障
- 可作为长期演进基础
