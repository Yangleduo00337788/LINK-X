## 2. 生产配置就绪

必填环境变量（见 `linkx-server/.env.prod.example` / `.env.prod`）：

| 变量 | 说明 |
|------|------|
| `SPRING_PROFILES_ACTIVE=prod` | 激活生产配置 |
| `JWT_SECRET` | ≥32 字节随机串 |
| `DB_HOST` / `DB_PORT` / `DB_NAME` / `DB_USERNAME` / `DB_PASSWORD` | MySQL（SSL） |
| `REDIS_HOST` / `REDIS_PORT` / `REDIS_PASSWORD` | Redis（生产无本机默认） |
| `MINIO_ENDPOINT` / `MINIO_ACCESS_KEY` / `MINIO_SECRET_KEY` / `MINIO_BUCKET_NAME` | 对象存储 HTTPS |
| `CORS_ORIGIN_1`（及可选追加项） | 前端域名白名单 |
| `MAIL_HOST` / `MAIL_USERNAME` / `MAIL_PASSWORD` / `MAIL_FROM` | 邮件（勿使用仓库默认值） |
| `IM_WS_PORT` | IM WebSocket 端口，默认 8081 |

启动前自检示例：

```powershell
# 缺任一必填则退出码非 0
$required = @('JWT_SECRET','DB_HOST','DB_USERNAME','DB_PASSWORD','REDIS_PASSWORD','MINIO_ACCESS_KEY','MINIO_SECRET_KEY','CORS_ORIGIN_1')
$missing = $required | Where-Object { -not [string]::IsNullOrWhiteSpace($_) -and [string]::IsNullOrWhiteSpace([Environment]::GetEnvironmentVariable($_)) }
if ($missing) { throw "缺少环境变量: $($missing -join ', ')" }
```

参考模板：`docs/env.production.example`（无真实密钥）。

## 3. 双端冒烟（自动化）

已覆盖（无需手工双开浏览器即可回归）：

| 场景 | 测试类 |
|------|--------|
| 送达 + 已读回执（双端 WS） | `DualClientLiveSmokeTest` |
| 拉黑后 WS 发送失败 / 解禁 | `DualClientLiveSmokeTest` |
| 入群审批申请→列表→通过 | `AlignmentGapSmokeTest` |
| 服务层已读/送达/拉黑 | `AlignmentGapSmokeTest` |

运行：

```bash
cd linkx-server
mvn -B "-Dtest=DualClientLiveSmokeTest,AlignmentGapSmokeTest" test
```

发版前仍建议用两个真实客户端点一遍 UI（开关屏蔽、群审批按钮）。

## 4. 已知边界（产品接受）

以下为**明确接受的边界**，不算阻塞上线：

| 边界 | 现状 | 本轮补强 |
|------|------|----------|
| 多人会议 | 管理房 + 本地预览，非完整 WebRTC mesh | 已补「双方入会 + offer/answer 信令」单测 |
| 桌面推送 | 本机 Notification，无 FCM/APNs | 保持边界；需移动端推送再立项 |
| 大群压测 | 集成冒烟非系统压测工具 | 扇出冒烟扩至 **40 人**并校验发送耗时 |
| 弱网 WebRTC | 策略单测 + Playwright Fake/CDP | 真实双端丢包仍为可选 |
| 前端覆盖率 | 最低门禁 | 可选抬升 |

验收结论：**在接受上表边界的前提下，IM 主链路可上线。**

## 5. 建议发版命令顺序

1. 执行/校验 SQL

4. 前端 `npm test` + `npm run test:e2e`  
5. 观察 `/api/health` 与 IM WS 端口
