- 应用级副作用（主题同步、自动登录、跨窗口监听）统一放在 `main.ts` 启动阶段，不在业务组件内直接操作 DOM 或 localStorage。
- Naive UI 全局 Provider 采用嵌套方式在 `AppRoot` 中一次性声明（`NConfigProvider` → `NMessageProvider` → `NDialogProvider`），组件内部不再重复包裹。
- 主题相关状态通过 `storeToRefs` 解构为响应式引用，并以计算属性派生 Naive UI 的 `theme` 与 `themeOverrides`，保持单一数据源。
- 首屏无关的大体积组件（如设置弹窗）使用 `defineAsyncComponent(() => import(...))` 异步懒加载，减小初始包体。
