本模块由三个入口文件构成自上而下的应用启动链：
- `main.ts` 是 Vite/Vue 的绝对入口，顺序执行：创建 Pinia 并注册 `pinia-plugin-persistedstate` → `createApp(AppRoot)` → `app.use(pinia/router)` → 读取持久化主题并通过 `applyDocumentTheme` 写入 `data-theme` → 监听 `localStorage` 变化实现多窗口主题联动 → 在路由切换前钩子中重新应用主题 → 最后 `app.mount('#app')`。
- `AppRoot.vue` 作为根组件，用 `<n-config-provider>` 包裹 Naive UI 全局 Provider（消息、对话框、中文本地化），通过计算属性根据 `appStore.theme` 选择暗色主题或跟随系统亮色，并集中注入 `GlobalThemeOverrides`；同时监听主题变化同步到 HTML 与 Electron 主进程，并在已登录且锁屏时通过 `<Teleport to="body">` 挂载 `LockScreen`。
- `App.vue` 仅作为 Router 出口，渲染当前路由页面并施加占满视口的样式。
依赖方向单向向下：`main.ts` → `AppRoot` → `App` → 各路由页面；所有应用级副作用（主题、自动登录）集中在 `main.ts` 与 `AppRoot`，页面组件不感知这些横切关注点。
