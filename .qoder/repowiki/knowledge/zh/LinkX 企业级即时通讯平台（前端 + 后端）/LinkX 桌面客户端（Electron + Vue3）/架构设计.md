本模块由两条并行子树组成：
- `electron/` 下的主进程（`main.ts`）负责创建 BrowserWindow、托盘、全局快捷键、多窗口（朋友圈、笔记编辑器）生命周期管理，并通过 `ipcMain.handle/on` 暴露 window 控制、自动启动、安全存储等 API；预加载脚本 `preload.cjs` 以 CJS 形式注入 `window.electronAPI`，避免 Vite ESM 在 Windows 上无法注入的问题。
- `src/` 下的渲染进程使用 `src/main.ts` 作为入口，统一创建 Pinia（含 persistedstate）、Vue Router、Naive UI 主题，并在挂载前读取持久化主题、执行自动登录；所有组件通过 `window.electronAPI` 调用主进程能力。
- 两端的契约集中在 `electron/preload.ts` 的类型声明与 `src/types/electron.d.ts` 中，IPC 通道名采用 `window:*` / `app:*` / `secure-storage:*` 命名空间，主进程在 `registerWindowIpc()` 中集中注册并支持热重载时 `removeHandler/removeAllListeners` 清理。
- 构建产物由 `vite.config.ts` 同时产出 `dist/`（渲染静态资源）与 `dist-electron/`（编译后的主进程），`package.json` 的 `main` 指向 `dist-electron/main/main.js`，`electron-builder` 打包为 NSIS/dmg/AppImage。
